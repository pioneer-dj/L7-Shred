**L7-Shred** — это сетевой протокол транспортного уровня (L4-over-UDP/L7) с динамической обфускацией, разработанный для противодействия системам глубокого анализа пакетов (DPI) и статистическому ИИ-анализу трафика. 

Этот документ предназначен для разработчиков, интегрирующих модуль `core` в сторонние VPN-клиенты, прокси-менеджеры или кастомные сетевые решения.

## 🧭 Как устроен протокол: Жизненный цикл пакета

Протокол работает по принципу "Матрёшки", где исходный IP-пакет (L3) полностью уничтожается как статистическая единица и пересобирается в поток фреймов маскирующего протокола.

### Схема конвейера (Pipeline)

 [ Входной IP-пакет из TUN ]
             │
             ▼
   1. [  Fragmentor  ]   ───► Нарезка на непредсказуемые куски (32–288 байт)
             │
             ▼
   2. [ AEAD Cipher  ]   ───► Шифрование (AES-256-GCM) + HMAC-SHA256 подпись
             │
             ▼
   3. [ Frame Parser ]   ───► Формирование кадра: [Token][EncLen][Payload][Padding]
             │
             ▼
   4. [  Mask Mixer  ]   ───► Обертка в L7-заголовки (VK, RuTube, WebRTC и т.д.)
             │
             ▼
   5. [TemporalJitter]   ───► Задержка по Гауссу (Бокс-Мюллер) + симуляция потерь
             │
             ▼
 [ Сетевой сокет (UDP) ] ───► Отправка удаленной стороне


 1. Формат кадра протокола (Frame)
Каждый пакет, уходящий в сеть, упаковывается структурой Frame (все системные поля в BigEndian):


Смещение(Байты)	             Поле	              Тип	                            Описание

0x00 - 0x1F	              AuthToken	            [32]byte	        HMAC-SHA256 подпись кадра для защиты от зондирования

0x20 - 0x21	               EncLen	             uint16	            Реальный размер зашифрованных полезных данных

0x22 - ...	               Payload	             []byte	            Зашифрованное тело пакета (данные TUN после фрагментации)

... - Конец	            Padding	                 []byte	        Криптографический мусор для подгонки кадра под размер L7-маски



🤝 Жизненный цикл сессии и Handshake
Перед обменом трафиком клиент и сервер обязаны установить сессию. Обмен хендшейками идет через чистый UDP (или транспортный слой) со строгой проверкой Nonce и временных меток во избежание Replay-атак.

Клиент                                     Сервер
  │                                          │
  ├────── HandshakeSyn (Nonce1, TS1) ───────►│ (Проверка TS, проверка Replay)
  │                                          │
  │◄───── HandshakeAck (Nonce2, TS2) ────────┤ (Генерация Session ID)
  │                                          │
  ├────── HandshakeFin (SessionEstablished) ─►│
  │                                          │
  ▼ ================== ТУННЕЛЬ ОТКРЫТ ================== ▼


  Ротация масок: Внутри сессии параллельно живут LocalMixer (исходящие маски) и RemoteMixer (входящие маски). Переключение пула разрешенных протоколов (ProtocolMode) происходит асинхронно по таймеру SwitchInterval.


📉 Специфика ReliableUDP (KCP) vs Маскирование
Текущая реализация ReliableUDP поверх kcp-go работает в режиме Stream. Это означает, что KCP может склеивать ваши замаскированные кадры. При написании кастомных масок учитывайте, что парсер Unwrap() на приемной стороне должен уметь вычленять границы фрейма по полю EncLen, а не надеяться, что один UDP-пакет равен одной маске
